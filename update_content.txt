are the files that the client receives for review, completion, printing, signing, or scanning.

12.4.1 Client Type Purpose
The selected client type determines:
ÔÇó which contract package is shown

12.5 Client Interface Requirements
The client interface must be much simpler than the Compliance interface.
The client should not see multiple fragmented compliance documents. Instead, the client
should receive the full contract package as one document.
12.5.1 Client Document Experience
The client interface must show:
ÔÇó one full contract package as a single received document
ÔÇó instructions to review the full package
ÔÇó instructions to print, sign, and scan where required
ÔÇó an upload area for scanned and signed files
12.5.2 Required Client Workflow
The client workflow should be:
1. Select client type: Trust, Foundation, Private Person, or Company
2. Open and review the full contract package
3. Follow the signing instructions
4. Upload scanned and signed documents
5. Submit documents for review
12.5.3 Client Upload Instructions
The client-facing upload page should contain clear instructions such as:
Instructions for Client Upload
Please review the full contract package carefully.
Print all pages that require a signature.
Sign where indicated.
Scan the signed pages clearly.
Upload the scanned and signed documents in PDF format.
Make sure all pages are complete and readable before submission.

12.7 Important Product Logic
The system should enforce these principles:
1. Compliance sees the full review process.
2. Relationship Managers only see the mandate ID and KYC corrections to follow up.
3. Clients only see the contract package, upload instructions, and signed-upload area.
4. Clients must choose whether they are a Trust, Foundation, Private Person, or Company.
5. Documents must always be separated into Blank and Signed categories.
